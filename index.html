2. PSTN Connectivity Options

Webex Calling provides flexible PSTN architecture deployment options to fit varying regional and infrastructure requirements.

  • Cisco Calling Plans: A fully managed cloud PSTN solution provided directly by Cisco. Includes procurement, porting, and billing directly via the Control Hub.
  • Cloud Connected PSTN (CCP): Route calling traffic through authorized, top-tier third-party telecommunication providers utilizing direct peering to the Webex Cloud.
  • Local Gateway (On-Premises PSTN): Leverages existing on-premises telecom trunks (SIP, T1/E1, PRI) via a Cisco Unified Border Element (CUBE) or approved local gateway router. This allows for smooth, phased migrations from legacy PBXs.

3. Network, Protocols, and Media Specifications

The underlying technical matrix relies on open-standard communication protocols optimized for enterprise-grade Quality of Service (QoS).

Protocols and Signaling

  • Signaling Protocol: Session Initiation Protocol (SIP) over TLS
  • Device Management: HTTPS, Next-Gen HTTP-based device activation codes, and TR-069/STUN for remote endpoints.
  • API Availability: Webhooks and extensive REST APIs via the Webex Developer Portal for third-party provisioning, call control, and administrative workflows.

4. Security, Compliance & Encryption

Security is embedded natively across the entire Webex signaling and media path infrastructure layout.

  • Data in Transit: Signaling is encrypted via TLS 1.2 or TLS 1.3. Media streams are encrypted using SRTP (Secure Real-time Transport Protocol) with AES-256 encryption keys.
  • Data at Rest: Call recordings, voicemail data, and user profile databases are encrypted via enterprise-managed or Cisco-managed keys using AES-256.
  • Identity & Authentication: OAuth 2.0, SAML 2.0 Single Sign-On (SSO), and native Multi-Factor Authentication (MFA) integrations via Cisco Duo.
  • Compliance Matrix: SOC 2 Type II, ISO/IEC 27001, HIPAA-ready, GDPR compliant, and FedRAMP authorized infrastructure status.

5. Device & Endpoint Ecosystem

Webex Calling supports a wide matrix of endpoints, allowing automated onboarding and remote firmware provisioning structures.

  • Cisco Multiplatform Phones (MPP): Fully supported series include the Cisco IP Phone 6800, 7800, 8800, and 9800 Series Desk Phones.

  • Third-Party SIP Devices: Support for validated third-party analog telephone adapters (ATAs) and standard compliance SIP endpoints.

6. Management, Diagnostics, and Analytics

Centralized provisioning and monitoring are executed entirely through the Cisco Webex Control Hub interface panel.

  • Unified Provisioning: Single-pane-of-glass interface for user onboarding, phone number mapping, auto-attendant configuration, and hunt group administration rules.
  • Real-time Analytics: Hop-by-hop call quality telemetry, network utilization metrics, and jitter/packet-loss monitoring dashboards.
  • Troubleshooting Tools: Granular end-to-end call leg analysis with Per-Call Audio Quality indicators to isolate local network vs ISP/cloud tier constraints.
  • Integration Ecosystem: Native hooks into ecosystem applications such as Salesforce, Jira, ServiceNow, and custom web API components via the Webex App Hub.
